# 飞书多维表格字段说明

本说明用于配合 [`github-feishu-template.xlsx`](./github-feishu-template.xlsx) 或 [`github-feishu-template.csv`](./github-feishu-template.csv) 使用。

## 推荐建表方式

优先级从高到低：

1. 直接复制现成的飞书多维表格模板
2. 新建空白多维表格后导入模板文件
3. 手动创建字段

## 推荐字段

| 字段名 | 推荐类型 | 建议保留 | 说明 |
| --- | --- | --- | --- |
| 标题 | 单行文本 | 是 | 仓库短标题 |
| 链接 | 单行文本 | 是 | GitHub 仓库链接，建议保留，便于去重 |
| 简介 | 多行文本 | 是 | 生成后的项目简介 |
| 分类 | 单行文本 | 是 | 项目分类，初始阶段建议先用文本 |
| 星标 | 数字 | 否 | GitHub Star 数 |
| Fork | 数字 | 否 | GitHub Fork 数 |
| 语言 | 单行文本 | 否 | 主语言 |
| Topics | 多行文本 | 否 | 标签列表 |
| 作者 | 单行文本 | 是 | 仓库 owner，建议保留，便于去重 |
| 仓库名 | 单行文本 | 是 | `owner/repo`，建议保留，便于去重 |
| 主页 | 单行文本 | 否 | 仓库 Homepage |
| 更新时间 | 单行文本 | 否 | 最近更新时间 |
| 来源 | 单行文本 | 否 | 默认写入 `GitHub` |

## 最低可用字段

至少保留这 5 列：

- `标题`
- `链接`
- `简介`
- `作者`
- `仓库名`

这样可以保证：

- 基本信息能够正常写入
- 去重更稳定
- 后续扩展字段时不需要重建整套流程

## 导入建议

1. 在飞书中创建空白多维表格
2. 选择“导入”或“从 CSV / Excel 导入”
3. 导入 [`github-feishu-template.xlsx`](./github-feishu-template.xlsx) 或 [`github-feishu-template.csv`](./github-feishu-template.csv)
4. 回到扩展“飞书”页，粘贴多维表格链接
5. 等待自动识别 `App Token`、`Table ID` 并读取字段
6. 检查字段映射是否全部命中

## 修改字段名时的建议

支持自定义字段名，但推荐采用以下两种方式之一：

1. 保持默认字段名，直接使用默认映射
2. 修改字段名后，在扩展“飞书”页读取字段并确认映射结果

当前扩展支持：

- 自动读取当前表字段
- 自动预匹配常见字段名
- 高亮未命中的映射项
- 写入时自动跳过当前表中不存在的列
