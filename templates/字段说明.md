# 飞书多维表格字段说明

这份说明配合 [`github-feishu-template.xlsx`](./github-feishu-template.xlsx) 或 [`github-feishu-template.csv`](./github-feishu-template.csv) 一起使用，适合发给第一次安装扩展的人。

## 推荐建表方式

优先级从高到低：

1. 直接复制一份你已经配好的飞书多维表格模板
2. 新建空白多维表格后，导入 [`github-feishu-template.xlsx`](./github-feishu-template.xlsx) 或 [`github-feishu-template.csv`](./github-feishu-template.csv)
3. 手动创建字段

如果是发给团队成员或朋友长期使用，最省心的是直接发“飞书模板复制链接”；如果不方便共享模板，就让对方导入 CSV。

## 推荐字段

| 字段名 | 推荐类型 | 是否建议保留 | 说明 |
| --- | --- | --- | --- |
| 标题 | 单行文本 | 是 | 仓库短标题 |
| 链接 | 单行文本 | 是 | GitHub 仓库链接，建议保留，便于去重 |
| 简介 | 多行文本 | 是 | AI 生成的项目简介 |
| 分类 | 单行文本 | 是 | 项目分类，首版建议先用文本 |
| 星标 | 数字 | 否 | GitHub Star 数 |
| Fork | 数字 | 否 | GitHub Fork 数 |
| 语言 | 单行文本 | 否 | 主语言 |
| Topics | 多行文本 | 否 | 标签列表 |
| 作者 | 单行文本 | 是 | 仓库 owner，建议保留，便于去重 |
| 仓库名 | 单行文本 | 是 | `owner/repo`，建议保留，便于去重 |
| 主页 | 单行文本 | 否 | 仓库 Homepage |
| 更新时间 | 单行文本 | 否 | 最近更新时间 |
| 来源 | 单行文本 | 否 | 默认写入 `GitHub` |

## 最低可用字段

如果对方只想先跑通，至少保留这 5 列：

- `标题`
- `链接`
- `简介`
- `作者`
- `仓库名`

这样可以保证：

- 基本信息能写入
- 去重更稳定
- 后面再扩列时不用重做整套流程

## 导入建议

1. 在飞书里新建空白多维表格
2. 选择“导入”或“从 CSV / Excel 导入”
3. 导入 [`github-feishu-template.xlsx`](./github-feishu-template.xlsx) 或 [`github-feishu-template.csv`](./github-feishu-template.csv)
4. 导入完成后，回到扩展“飞书”页
5. 粘贴多维表格链接，自动识别 `App Token` 和 `Table ID`
6. 点击“读取飞书字段”
7. 检查字段映射是否已自动命中

## 如果改了字段名

可以改，但建议满足下面两种之一：

1. 保持默认字段名不变，直接零配置使用
2. 改成自己的字段名后，在扩展“飞书”页读取字段并确认字段映射

扩展已经支持：

- 自动读取当前表字段
- 自动预匹配常见字段名
- 高亮未命中的映射项
- 写入时自动跳过当前表里不存在的列

但如果要给别人长期使用，仍然建议默认字段名保持一致，这样最省沟通成本。
