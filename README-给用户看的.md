# GitHub 项目采集扩展使用说明

这是一款浏览器扩展，用来把当前 GitHub 仓库的信息一键整理后写入飞书多维表格。

## 你会得到什么

- 在 GitHub 仓库页右下角看到采集按钮
- 点一次即可把仓库标题、链接、简介、分类、星标等信息写入飞书
- 支持自动读取飞书字段并完成字段映射

## 安装扩展

1. 打开 Chrome 或 Edge 的扩展管理页
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择扩展目录：[`extension`](./extension)

## 先准备飞书多维表格

推荐二选一：

1. 直接复制一份现成的飞书模板
2. 新建空白多维表格后，导入下面任一模板文件

模板文件：

- [`templates/github-feishu-template.xlsx`](./templates/github-feishu-template.xlsx)
- [`templates/github-feishu-template.csv`](./templates/github-feishu-template.csv)

如果不清楚字段含义，可以看：

- [`templates/字段说明.md`](./templates/%E5%AD%97%E6%AE%B5%E8%AF%B4%E6%98%8E.md)

## 首次配置

打开扩展弹窗后，按这个顺序操作：

1. 在“模型”页填写模型接口地址、API Key、模型名称
2. 在“飞书”页填写 `App ID`、`App Secret`
3. 粘贴多维表格链接，让扩展自动识别 `App Token` 和 `Table ID`
4. 点击“读取飞书字段”
5. 回到“采集”页，测试模型和飞书连接

## 开始采集

1. 打开任意 GitHub 仓库主页
2. 点击右下角悬浮按钮，或打开扩展弹窗点击“保存当前项目”
3. 等待写入完成

## 常见问题

### 1. 飞书里删掉记录后，为什么还能提示重复？

当前版本会在本地记住最近采集过的项目，但再次采集时也会复核飞书真实记录。如果你刚删除过旧记录，重新采集一次即可自动刷新本地缓存。

### 2. 为什么有些字段没写进去？

通常是因为当前多维表格里没有对应列。建议先导入模板，再在扩展里点击“读取飞书字段”。

### 3. 知识库链接能直接用吗？

可以。扩展支持识别知识库链接，并会尝试继续解析真实的多维表格 `App Token`。

## 需要准备的配置

- 模型接口地址
- 模型 API Key
- 模型名称
- 飞书 App ID
- 飞书 App Secret

可选：

- GitHub Token

## 提示

- 纯扩展模式会把密钥保存在浏览器扩展本地存储中
- 更适合个人设备或受信设备使用
- 如果是团队长期共用，建议改成服务端方案
